# AI Auto Test SOP

**Universal, Agent-Agnostic, Self-Evolving AI Testing Framework.**

[中文版 README](README_CN.md) | [CHANGELOG](CHANGELOG.md)

## 📖 Overview

**AI Auto Test SOP** is a standardized, AI-driven automated testing framework.
Unlike traditional testing scripts that rely on specific tools or rigid workflows, this SOP is:

*   **Spec-Agnostic**: Accepts specs from OpenSpec, Yuque, Markdown, or even natural language.
*   **Agent-Agnostic**: Automatically adjusts its execution strategy based on the AI Agent's capabilities (Qoder, Cursor, Aone, Claude Code, Hermes, etc.).
*   **Self-Evolving**: Builds a knowledge base of pitfalls and adaptations from real execution.

## 🏗️ Architecture: Framework vs Workspace

This SOP enforces a **clean separation** between the universal framework and project-side accumulation:

```
your-project/
├── .test-sop/              ← Universal framework (read-only, updated via git pull)
└── .test-workspace/        ← All project-side files (config + accumulation + outputs)
    ├── config.yaml         ← Project configuration
    ├── adaptations.yaml    ← Tier-1 evolution (auto-tuned parameters)
    ├── memory.md           ← Team preferences & project context
    ├── skills/             ← Reusable success workflows (auto-accumulated)
    ├── pitfalls/           ← Project-specific pitfalls
    └── runs/<req-id>/      ← Per-requirement test artifacts
```

> **Key principle**: The framework is a SPECIFICATION. Your team's accumulation lives entirely in `.test-workspace/` and never leaks back into `.test-sop/`.

## 🏁 Quick Start

### ⚡ Option A: Auto-Bootstrap (Recommended)
*The simplest way to let the AI handle everything.*

1. **Install Framework + Workspace**:
   ```bash
   git clone https://github.com/xuwu0/ai-auto-test-sop.git .test-sop
   bash .test-sop/install.sh
   ```
   This creates `.test-workspace/` with default config, adaptations, memory, and empty skills/pitfalls/runs directories.

2. **Activate Agent**:
   Copy the `INSTRUCTIONS.md` to your project root (or paste it into your AI's Custom Instructions):
   ```bash
   cp .test-sop/INSTRUCTIONS.md .
   ```
   Now type `/test-sop <requirement>` and the AI will auto-bootstrap, self-check, and run.

### 🔧 Option B: Manual Setup

1. `git clone https://github.com/xuwu0/ai-auto-test-sop.git .test-sop`
2. Edit `.test-workspace/config.yaml` (auto-generated by install.sh) to match your stack.
3. Ask your AI to execute `/test-sop <requirement-source>`.

### 🔌 MCP Configuration (Required for Full-Auto Mode)

Edit `.test-workspace/config.yaml`:
```yaml
mcp:
  enabled_capabilities:
    - cap.log.query        # for L2 Log validation
    - cap.database.query   # for L3 Data validation
    - cap.deploy.async     # for deployment
```
> **💡 No MCP tools?** Set `execution_mode: assisted`. The AI will generate a Manual Guide for you.

## 🔄 Updating

The framework is under active development. Update via:
```bash
cd .test-sop && git pull origin main
```
Your `.test-workspace/` is untouched.

## 📺 Monitoring & Transparency

Track AI workflow progress via:

1. **`.test-workspace/runs/<id>/test-status.json`** — The "Dashboard". Check `current_step` and `retry_count`.
2. **`.test-workspace/runs/<id>/execution-log.md`** — The "Black Box". Real-time audit of every RPC call, SQL query, and Shell command.
